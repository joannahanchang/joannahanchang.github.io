Week 7:

  • This week, I learned about Bootstrap JavaScript components, which make use of JavaScript for their functionality, but do not require to be written in JavaScript. We could use HTML5 custom data-* attributes that Bootstrap has defined in their code to access the JavaScript functionality.

  • We reviewed JavaScript this week.
  • The 8 types of JavaScript as of the most recent ECMAScript standard are: number, string, boolean, undefined, null, object, symbol, and BigInt.
  • We went over let vs. const. var, truthy vs. falsy, JavaScript operators, if/elseif/else, switch statement, etc.

Week 8:

  • I learned that the JavaScript used in Bootstrap is through a popular JS library called jQuery.
  • A couple examples of using jQuery is changing the carousel cycle interval and add play/pause functionality on the buttons, and modifying jQuery script so that when the button is clicked, it will change from play to pause or vice versa.
  • We learned about Sass as the CSS preprocessor that is used to write the source CSS code for Bootstrap 4 and it is useful if we want to theme Bootstrap or customize its source. Overall, CSS preprocessors can make it easier to write and maintain large, complex stylesheets.
  • There are two variants of Sass - Sass (.sass) and Scss (.scss) - there are a few syntactic differences but their functionality is the same.
  • I practiced writing Scss codes to incorporate variables and nesting.
  • I installed the Node Scss package to compile .scss to .css.
  • I learned about NPM scripts, which are basically scripts to automate repetitive tasks, delete temporary files and folders, etc.
  • I added an NPM script in package.json to automate compilation. Specific thing I did include: moving jQuery script for the carousel to an external JavaScript file, installing the onchange module to enable my project to watch for changes to its files, installing the parallel shell module to enable my project to run multiple NPM scripts in parallel, and watching for any changes to my Sass file and automatically compile it to css.
  • With regard to automating the process of building a distribution folder by writing NPM scripts: stored copies of all font-awesome icon files, compressed and minified images, minified and uglified code files, cleaned up the distribution folder, and generated the distribution files with an NPM command.
  • We reviewed JavaScript and the use of while loops and do...while loops, for loops, and arrays.

Week 9:

  • This week, we reviewed JavaScript.
  • We reviewed how to write and use JavaScript function declarations. Function hoisting means that before any JavaScript is executed, the code is first scanned for function declarations and any that are found are "hoisted" to the top of their scope. This means that we can use functions that were created with function declarations before we get to them in the code.
  • We reviewed the use of events in JavaScript, primarily onevent handlers and event listeners.
  • We learned the use of JavaScript function expressions, or ways to write functions inline in expressions. This can be useful including being able to use a function as an argument for another function (which is known as a callback function).
  • We also learned about the differences between anonymous and named function expressions, as well as that function expressions are not hoisted.
  • We learned about the new ES6 arrow functions, which are a more compact version of function expressions, are frequently used as callback functions, and will be used very frequently in React.
  • We reviewed the differences between let vs. const vs. var.
  • Code Challenges: There were 3 code challenges this week, which can be found in my CodePen: JS - Arrow Function Writing Drills, JS - MicroTweet, and JS - Fix Me.
  • Project: I learned about a programming algorithm called binary search and used my Bootstrap and JavaScript knowledge to improve a number-guessing game using binary search, update the Bootstrap UI, and write JavaScript to add a reset button, as well as fix broken behavior. The project is saved in my Codepen: JavaScript Binary Search Number Guessing Game.
